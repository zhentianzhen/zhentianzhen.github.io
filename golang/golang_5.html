使用mgo时犯了个错，找了半天，谨记！

      程序运行一段时间后，和mongo挂钩的接口都不响应了。开始以为进程挂了，然而并没有，又猜测是mongo挂了，然而也没有，mongo连接数也不是很多。只好先重启进程，重启后什么都好了，但是必须找出原因。后来终于发现，原来是这段代码的问题。。。


      代码如下：


m := Instance()
session, err := m.GetSession()
if err != nil {
  log.Println(err)
  return err
}

      上面代码的问题在于打开一个session过后，使用完时并没有关闭它，这就造成随着时间增加，使用次数增加，打开的session数会过多，因此只需要加一行代码即可搞定：


m := Instance()
session, err := m.GetSession()
if err != nil {
  log.Println(err)
  return err
}
defer session.Close()
